# Bug 练习提示 - 数据处理器

本文档列出了 `data_processor_*.py` 中所有可改进的问题点，但**不提供答案**。
请使用这些提示进行调试、测试和代码重构练习。

---

## 🔴 严重问题 (Critical)

### 1. 资源泄露
- **位置**: `process_file_lines()` 函数
- **提示**: 文件打开后没有正确关闭，即使发生异常也不会关闭
- **影响**: 文件描述符泄露，可能导致 "too many open files" 错误
- **测试方法**: 在循环中调用此函数数百次，观察系统资源

### 2. 可变默认参数陷阱
- **位置**: `load_config()` 函数
- **提示**: 默认参数 `cache={}` 是可变对象
- **影响**: 多次调用函数会共享同一个字典，导致意外行为
- **测试方法**: 连续调用两次，修改返回的字典，再调用第三次检查是否被污染

### 3. 除零错误
- **位置**: `calculate_average()` 函数
- **提示**: 当传入空列表时会发生什么？
- **影响**: `ZeroDivisionError`
- **测试方法**: 传入 `[]` 空列表

### 4. 索引越界
- **位置**: `find_max_value()` 函数
- **提示**: 当传入空列表时，第一行代码会发生什么？
- **影响**: `IndexError`
- **测试方法**: 传入 `[]` 空列表

## 🟠 中等问题 (Major)

### 5. 缺少异常处理
- **位置**: `process_file_lines()` 函数
- **提示**: 
  - 文件不存在时会怎样？
  - 行格式不正确时会怎样？
  - 转换整数失败时会怎样？
- **影响**: 程序崩溃，没有友好的错误信息
- **测试方法**: 传入不存在的文件路径，或格式错误的文件

### 6. 边界条件错误
- **位置**: `process_batch()` 函数
- **提示**: 当列表长度不是 batch_size 的整数倍时，最后一批数据会怎样？
- **影响**: 数据丢失
- **测试方法**: 传入 10 个元素，batch_size=3，检查结果数量

### 7. 输入验证缺失
- **位置**: `calculate_discount()` 函数
- **提示**: 
  - 折扣百分比为负数时会怎样？
  - 折扣百分比超过 100 时会怎样？
  - 价格为负数时会怎样？
- **影响**: 产生不合理的计算结果
- **测试方法**: 传入 discount_percent=150 或 -10

### 8. 浅拷贝问题
- **位置**: `copy_list()` 函数
- **提示**: 如果列表包含嵌套列表，修改拷贝后的列表会影响原列表吗？
- **影响**: 意外的数据修改
- **测试方法**: 传入 `[[1,2], [3,4]]`，修改拷贝后的嵌套元素

### 9. 缺少键存在检查
- **位置**: `get_nested_value()` 函数
- **提示**: 如果某个 key 不存在会发生什么？
- **影响**: `KeyError` 异常
- **测试方法**: 传入不存在的 key 路径

### 10. None 值处理
- **位置**: `compare_strings()` 函数
- **提示**: 如果任一参数是 None 会怎样？
- **影响**: `AttributeError`
- **测试方法**: 传入 None 作为参数

## 🟡 轻微问题 (Minor)

### 11. 代码重复 (DRY 原则)
- **位置**: `merge_lists()` 函数
- **提示**: 两段几乎相同的代码，能否提取为辅助函数或使用更简洁的方法？
- **影响**: 代码维护困难
- **改进方向**: 寻找 Python 内置的集合操作

### 12. 性能问题 - 列表查找
- **位置**: `merge_lists()` 函数
- **提示**: `if item not in result` 的时间复杂度是多少？对于大列表会怎样？
- **影响**: O(n²) 时间复杂度
- **改进方向**: 考虑使用集合 (set) 数据结构

### 13. 性能问题 - 线性搜索
- **位置**: `search_item()` 函数
- **提示**: 如果这个函数被频繁调用，有什么优化方案？
- **影响**: 每次搜索都是 O(n)
- **改进方向**: 如果数据有序或需要频繁搜索，考虑其他数据结构

### 14. 浮点数精度问题
- **位置**: `format_currency()` 函数
- **提示**: 尝试 `format_currency(19.999)` 或涉及浮点运算的结果
- **影响**: 显示金额可能不准确
- **测试方法**: 传入 `0.1 + 0.2` 或 `19.999`

### 15. 不完整的邮箱验证
- **位置**: `validate_email()` 函数
- **提示**: 哪些无效邮箱会被错误地接受？
- **影响**: 接受格式错误的邮箱地址
- **测试方法**: 测试 `"@"`, `"@."`, `"test@"`, `"test@com"` 等

### 16. 潜在的安全问题
- **位置**: `read_user_data()` 函数
- **提示**: 如果 `user_id` 来自用户输入，直接拼接到查询字符串有什么问题？
- **影响**: SQL 注入风险
- **改进方向**: 参数化查询

### 17. 时区问题
- **位置**: `get_timestamp()` 函数
- **提示**: `datetime.now()` 返回的是本地时间还是 UTC？在不同时区运行会怎样？
- **影响**: 时间戳不一致，日志分析困难
- **改进方向**: 使用 UTC 时间并明确时区

- INFO 级别：关键操作

- batch_size
- 验证规则
- 超时时间

使用 profiling 工具：
```bash
python -m cProfile data_processor_*.py
```
找出瓶颈并优化，记录优化前后的性能对比。

### 挑战 6: 添加 CLI 接口
使用 argparse 添加命令行接口：
```bash
python data_processor.py --input file.csv --output result.json
```
